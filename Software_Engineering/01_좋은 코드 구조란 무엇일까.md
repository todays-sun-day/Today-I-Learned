# [MVC 패턴] Model - View - Controller

## 1. MVC 패턴이란?

소프트웨어 공학에서 사용되는 디자인 패턴으로, 애플리케이션을 **Model, View, Controller** 세 가지 역할로 나누어 코드를 구성하는 방식이다.

* **Model (모델)**: 데이터와 비즈니스 로직 (주방, 레시피)
* **View (뷰)**: 사용자에게 보이는 UI (레스토랑 홀, 메뉴판)
* **Controller (컨트롤러)**: Model과 View를 연결하는 중재자 (웨이터)

## 2. 왜 사용하는가?

**관심사 분리 (Separation of Concerns)** 를 통해 코드의 **유지보수성**과 **재사용성**을 높이기 위함이다.

1.  **유지보수:** UI 수정은 View만, 로직 수정은 Model만 보면 된다.
2.  **협업:** 프론트엔드(View)와 백엔드(Model/Controller)가 작업을 분리하기 용이하다.
